## 1. Python 环境搭建

### 方式一：下载安装包（推荐）

**下载地址**：https://www.python.org/downloads/

1. 点击 **Download Python 3.x.x** 下载安装包
2. 运行安装包，**务必勾选** `Add Python to PATH`
3. 打开命令行验证：
```bash
python --version
pip --version
```

### 方式二：使用 conda

```bash
# 如果有 conda 环境，直接创建新环境
conda create -n xuexitong python=3.9
conda activate xuexitong
```


## 4. 下载浏览器驱动

### Chrome 浏览器

1. 查版本：打开 Chrome → `设置` → `关于 Chrome`
2. 下载对应版本驱动：https://googlechromelabs.github.io/chrome-for-testing/
3. 解压得到 `chromedriver.exe`
4. 放在项目目录下，记住路径

### Edge 浏览器

1. 查版本：打开 Edge → `设置` → `关于 Microsoft Edge`
2. 下载对应版本驱动：https://developer.microsoft.com/zh-cn/microsoft-edge/tools/webdriver/
3. 解压得到 `msedgedriver.exe`
4. 放在项目目录下，记住路径

## 6. 配置说明

### 6.1 首次使用配置

运行 `python start.py` 后，进入 **设置页面** 填写：

| 配置项 | 说明 |
|--------|------|
| 浏览器 | 选择 chrome 或 edge |
| 驱动地址 | 点击"选择文件"找到刚才下载的驱动 |

## 7. 常见问题

### Q1: 提示 `PermissionError`

**原因**：没有管理员权限

**解决**：关闭程序后，右键点击 `start.py` 或 `start.exe`，选择 **"以管理员身份运行"**

### Q2: 浏览器驱动版本不匹配

**错误信息**：`This version of ChromeDriver only supports Chrome version xxx`

**解决**：
1. 确认浏览器版本：`chrome://settings/help`
2. 下载对应版本的驱动
3. 更新设置里的驱动路径

### Q3: 提示 `请检查网络连接`

**原因**：网络问题或代理冲突

**解决**：
1. 关闭代理/VPN
2. 检查网络是否正常
3. 重试

### Q4: 页面无法识别

**原因**：学习通网页结构可能已更新

**解决**：这是脚本失效的信号，可以：
1. 等待作者更新脚本
2. 提交 Issue 给作者
3. 自行分析网页结构修改代码

### Q5: 人脸认证无法跳过

**解决**：
1. 确保设置里勾选了"跳过人脸"
2. 第一次手动完成人脸认证
3. 查看 `task/tool/face_cookies.pkl` 是否生成

### Q6: 倍速太高任务点被清空

**解决**：使用 1x 或 2x 倍速，不要用 16x

### Q7: 安装包下载慢

**解决**：使用国内镜像
```bash
pip install xxx -i https://pypi.tuna.tsinghua.edu.cn/simple
```
